1. 데이터셋 소개

본 과제에서는 머신러닝 입문 과정에서 가장 대표적으로 사용되는 Titanic Dataset을 선택하였다. 1912년 타이타닉호 침몰 사고 당시 승객 정보와 생존 여부를 담은 데이터로, 총 891명의 승객에 대해 15개의 컬럼이 제공된다. 결측치, 범주형 변수, 수치형 변수가 모두 포함되어 있어 특성 공학 학습에 매우 적합한 데이터셋이다.

2.9. EDA 요약

  • 데이터 품질: deck(77%), age(19.9%) 컬럼에 결측치 존재 → 처리 전략 비교 필요
  • 이상치: fare, parch 변수에 다수의 이상치 → RobustScaler 활용 가치 있음
  • 불균형: 사망:생존 ≈ 6:4, 약한 불균형 → F1-score / ROC-AUC 추가 평가
  • 주요 변수: sex, pclass, fare가 생존 여부와 강한 관계
  • 중복 변수: class↔pclass, embark_town↔embarked, alive↔survived → 학습 시 제거 필요

3. 특성 공학 파이프라인 구현

3.1. 파생 변수 생성

도메인 지식을 활용하여 다음 4개의 파생 변수를 생성하였다.

파생 변수생성 공식의미
family_sizesibsp + parch + 1본인 포함 가족 수
fare_per_personfare / family_size1인당 운임 (가족 단위 가격 보정)
age_groupage 구간화 (5단계)child / teen / young_adult / adult / senior
is_childage < 12어린이 여부 (우선 구조 효과 반영)

family_size가 2~4명일 때 생존율이 가장 높다(약 55~72%). 혼자(1명) 탑승하거나 5명 이상의 대가족일 경우 생존율이 급격히 떨어진다. 또한 어린이(is_child=1)의 생존율(58%)이 성인(38%)보다 1.5배 높아 여성과 어린이 우선 구조 원칙이 데이터에 명확히 드러난다.

(1) Base 실험의 극단적으로 낮은 성능

Base 실험(F1=0.4615)은 다른 실험들보다 약 30%p 이상 낮은 성능을 보였다. 이는 (1) 결측치를 그대로 사용하지 못해 수치형 변수만 사용했고, (2) 가장 중요한 변수인 sex(성별)와 age(나이)를 사용하지 못했기 때문이다. EDA에서 확인된 것처럼 sex 변수의 영향이 압도적이므로 이를 활용하지 못한 영향이 크다. 이는 특성 공학(Feature Engineering)이 모델 성능에 결정적으로 기여함을 명확히 보여준다.

(2) Exp-1 (Mean + One-Hot + Standard)이 1위

F1=0.7786으로 가장 우수했다. 전체 25개 변수를 모두 사용하여 정보 손실이 없었고, One-Hot Encoding이 Logistic Regression과 잘 맞았다. Standard 스케일링은 Logistic Regression의 가정(정규분포 가까운 입력)과 일치하여 좋은 결과를 냈다.

(3) Exp-3 (Most Frequent + One-Hot + Robust + FS)이 2위

F1=0.7634로 Exp-1과 큰 차이가 없다(약 1.5%p). 변수를 10개로 줄였음에도 거의 동일한 성능을 보였다. RobustScaler는 이상치에 강하나 본 데이터의 LogReg에서는 Standard와 큰 차이가 없었다. 모델 단순화 측면에서는 Exp-3이 매우 좋은 선택이다.

(4) Exp-2 (Median + Label + MinMax + FS)가 3위

F1=0.7442로 다른 실험들보다 약간 낮다. 가장 큰 원인은 Label Encoding이다. Logistic Regression은 선형 모델이므로 순서가 없는 범주(예: embarked: C=0, Q=1, S=2)에 임의의 순서를 부여하는 Label Encoding이 적합하지 않다. 단, ROC-AUC는 0.8709로 가장 높아 임계값 조정 시 다른 결과가 나올 수도 있다.

7.2. Permutation Importance — Feature Importance 시각화 고도화

8. 최종 결론

8.1. 과제 핵심 질문에 대한 답변

① 어떤 전처리 전략이 가장 효과적이었는가?

본 실험에서는 Median + One-Hot Encoding + StandardScaler 조합이 18개 조합 중 최고의 F1-score(0.7879)를 달성했다. 실험 비교(Exp-1 ~ Exp-3)에서도 One-Hot + Standard 조합인 Exp-1이 우수했다. 한편 18개 조합 평균에서는 RobustScaler가 가장 안정적이었다. 결론적으로 데이터에 이상치가 많을 때는 RobustScaler, 일반적으로는 StandardScaler가 좋은 선택이다.

② One-Hot Encoding이 항상 좋은가?

본 실험에서는 One-Hot이 Label Encoding보다 평균 F1-score가 약 1%p 더 높았다. Logistic Regression처럼 선형 모델은 변수 간 순서가 없는 범주형에 임의 정수를 부여하는 Label Encoding과 잘 맞지 않는다. 하지만 트리 기반 모델(RF, XGBoost)에서는 Label Encoding도 잘 작동할 수 있고, 카테고리가 수십~수백 개인 경우 One-Hot은 차원이 폭발적으로 늘어나 오히려 성능이 떨어질 수 있다. 즉 모델 종류와 변수 카디널리티에 따라 달라진다.

③ Feature Selection이 과적합 감소에 기여했는가?

변수 선택 전(F1=0.7879) → 후(F1=0.7634)로 절대 성능은 약간 하락했지만, 변수 수를 25 → 10으로 60% 축소했음에도 하락은 약 2.5%p에 불과했다. 이는 모델 복잡도 감소, 학습 시간 단축, 해석 가능성 향상에 큰 이점을 가진다. 본 데이터(891개)는 과적합이 심각하지 않아 효과가 두드러지지 않았지만, 변수 수가 많거나 샘플 수가 적은 데이터에서는 과적합 감소 효과가 더 크게 나타날 것이다.

④ 스케일링이 모델별로 어떤 영향을 미쳤는가?

본 실험에서는 Logistic Regression만 사용했지만, 일반적으로 거리 기반/선형 모델 (LogReg, SVM, KNN)은 스케일링에 민감하고, 트리 기반 모델(RF, XGBoost, LightGBM)은 스케일링에 영향받지 않는다. 본 데이터에서는 fare 변수의 이상치가 많아 RobustScaler가 평균적으로 가장 우수했다 (F1=0.7806). 이상치가 있는 변수가 있으면 평균/표준편차에 영향이 가는 StandardScaler보다 RobustScaler가 권장된다.

⑤ Feature Engineering이 실제 성능 향상에 얼마나 기여했는가?

Base 실험(전처리/FE 없음) F1=0.4615 → Exp-1(전체 FE 적용) F1=0.7786으로 약 68.7% 상승했다. 또한 파생 변수 fare_per_person이 모든 변수 중 Feature Importance 1위(0.1597)를 차지했다는 사실은 도메인 지식을 반영한 파생 변수가 원본 변수보다 더 강력한 예측력을 가질 수 있음을 보여준다. 특성 공학은 모델 자체보다 더 큰 영향을 미치는 핵심 단계임이 확인되었다.

  • 최고 단일 조합: Median + One-Hot + StandardScaler (F1=0.7879)
  • 최고 단일 모델: Random Forest (F1=0.7704), ROC-AUC는 Logistic Regression이 1위(0.8725)

  • 인코딩: 선형 모델에는 One-Hot이 Label보다 평균 1%p 우수
  • 스케일링: 이상치가 많은 fare가 있어 RobustScaler가 평균적으로 가장 안정적
  • 변수 선택: Top 10 변수만으로도 전체 25개 대비 96.8%의 F1-score 유지
  • GridSearchCV: Best CV F1=0.7830 (RandomForest, max_depth=8)

  • 설명 가능성: SHAP/LIME/Permutation 모두 sex, adult_male, pclass를 핵심 변수로 일관되게 지목

8.3. 한계 및 발전 방향

  • 본 실험은 단일 train/test 분할에 의존하여 결과의 분산을 충분히 평가하지 못했다. Stratified K-Fold Cross Validation으로 확장하면 더 신뢰성 있는 비교가 가능하다 (GridSearchCV 부분에서 일부 반영).
  • GridSearchCV로 RF의 하이퍼파라미터를 일부 튜닝하였으나, Logistic Regression의 정규화 강도 외 추가적인 모델별 세밀한 튜닝은 한계가 있었다. Optuna 등 Bayesian Optimization을 활용하면 더 효율적인 탐색이 가능하다.
  • SHAP과 LIME으로 모델 해석을 수행했으나, 본 보고서에서는 일부 결과만 정리했다. 개별 예측에 대한 심층 분석(특히 오분류 케이스)을 확장하면 모델 개선의 단서를 얻을 수 있다.
  • 이름(name)에서 호칭(Mr/Mrs/Miss/Master)을 추출하거나 cabin 번호에서 위치 정보를 추출하는 등 더 풍부한 파생 변수를 생성하면 성능이 더 향상될 가능성이 있다.
  • 클래스 불균형(62:38)이 심각한 수준은 아니나, SMOTE 등 오버샘플링 기법이나 임계값 최적화로 소수 클래스(생존)의 Recall을 더 높일 여지가 있다.
